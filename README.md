# 3D Pac-Man — Наклонная платформа

Pac-Man на наклонной платформе с перспективной 3D-проекцией.  
Управление — не кнопками движения, а физическим наклоном доски.  
Пакман катится сам под действием гравитации.

## Запуск

```bash
git clone <repo>
cd pacman3d
go mod init pacman3d
go get github.com/hajimehoshi/ebiten/v2
go run .
```

Требуется Go 1.21+

## Управление

| Клавиша | Действие |
|---|---|
| `←` | Наклонить доску влево |
| `→` | Наклонить доску вправо |
| `↑` | Наклонить доску от себя (пакман катится вперёд) |
| `↓` | Наклонить доску на себя (пакман катится назад) |
| `R` | Рестарт |

Кнопки не управляют пакманом напрямую — они меняют угол платформы.  
Пакман движется туда, куда наклонена доска, под действием гравитации.

## Механика физики

```
tiltZ (←→) → sin(tiltZ) × gravity → ускорение по X
tiltX (↑↓) → sin(-tiltX) × gravity → ускорение по Z
```

На каждом кадре:
1. Клавиши изменяют угловую скорость наклона (`tiltVX`, `tiltVZ`)
2. Угловая скорость затухает при отпускании кнопки (коэффициент `tiltDamp = 0.86`)
3. Углы ограничены диапазоном `±0.32 рад (~18°)`
4. Из углов вычисляется ускорение пакмана через проекцию гравитации
5. Скорость умножается на `friction = 0.85` каждый кадр
6. При столкновении со стеной скорость отражается с коэффициентом `0.25`

## Камера и проекция

Камера фиксирована — стоит высоко над центром доски и смотрит вниз под углом ~56°.

```
eye = (boardCX, 500, boardCZ + 340)
target = (boardCX, 0, boardCZ)
```

Базис камеры строится через `forward`, `right`, `up` векторы.  
Применяется перспективная проекция с фокусным расстоянием `camFovD = 420`.

Порядок трансформаций для каждой точки:
```
Мировые координаты
  → Наклон доски (вращение вокруг X и Z осей вокруг центра доски)
  → Перевод в пространство камеры (dot product с базисом)
  → Перспективное деление (x/z, y/z)
  → Экранные координаты
```

## Рендеринг

Используется **Painter's Algorithm** (сортировка от дальнего к ближнему).

Каждая ячейка лабиринта генерирует набор `renderItem` с глубиной.  
Стена рисуется 4 гранями: верхняя, передняя, правая, задняя.  
Пол — шахматный паттерн из двух оттенков для глубины восприятия.

Треугольники рисуются через `screen.DrawTriangles` с белым 1×1 субизображением  
как источником — цвет задаётся через `ColorR/G/B/A` вершин.

## Параметры настройки

| Константа | Значение | Описание |
|---|---|---|
| `cellSize` | 32 | Размер ячейки лабиринта в пикселях |
| `wallH` | 22 | Высота стен |
| `maxTilt` | 0.32 рад | Максимальный угол наклона (~18°) |
| `tiltSpeed` | 0.013 | Скорость изменения наклона |
| `tiltDamp` | 0.86 | Затухание наклона при отпускании |
| `grav` | 380 | Сила гравитации (пикс/с²) |
| `friction` | 0.85 | Трение (множитель скорости/кадр) |
| `camH` | 500 | Высота камеры над доской |
| `camD` | 340 | Дистанция камеры позади центра |

## Структура кода

```
main.go
├── init()              белый пиксель для DrawTriangles
├── project()           3D → 2D проекция с учётом наклона
├── drawTri/drawQuad    примитивы рисования
├── Ghost               структура призрака + ИИ
├── Game
│   ├── initMaze()      парсинг лабиринта
│   ├── updateTilt()    обработка клавиш → углы наклона
│   ├── updatePacman()  физика + сбор пелетов
│   ├── updateGhosts()  ИИ + коллизия с пакманом
│   ├── Update()        главный игровой цикл
│   └── Draw()          рендеринг (painter's algorithm)
└── main()
```

## Лабиринт

Лабиринт задаётся в виде среза строк `maze []string` (20×21).

| Символ | Значение |
|---|---|
| `#` | Стена |
| `.` | Пелет (+10 очков) |
| `o` | Большой пелет (+50 очков) |
| `P` | Стартовая позиция пакмана |
| ` ` | Пустое пространство |
